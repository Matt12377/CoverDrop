# 华语音乐库文件名全量审计

## 结论

本报告不是抽样结果。审计程序按快照中的专辑顺序和专辑内音频顺序遍历了全部 **4,867 张专辑、68,525 个音频文件名**，并把每条 `relativePath` 依次加入 SHA-256。完整序列摘要为：

```text
a9fff44eab0c7259e51d314f09ac66cf089c2ebf7963b8e09316824e28541b98
```

输入文件虽然使用 `.db` 后缀，实际是 CoverDrop schema v1 的旧 JSON 扫描快照，不是 SQLite。审计使用项目的旧快照结构解码，未修改数据库或音乐库。

根据全部文件名、目录名和标签的交叉验证，应清洗的字段只有派生展示/搜索用的 `artistName` 与 `albumName`。真实目录路径、音频文件名、`relativePath`、原始音频标签、快照内容和封面保存位置都不应被清洗规则改写。

## 专辑目录噪声总结

4,867 个原始专辑目录名中，有 4,458 个至少命中一种高置信结构噪声。类别会重叠：

| 原始噪声类别 | 专辑数 | 典型形态 |
| --- | ---: | --- |
| 格式、来源、压制或版本标记 | 3,291 | `[WAV]`、`[Qobuz]`、`SACDISO`、`24bit 96kHz` |
| 前置日期或年份 | 2,903 | `1999-标题`、`2023.04.18 标题` |
| 目录中重复歌手名 | 1,606 | `年份-歌手-标题`、`歌手《标题》` |
| 前置来源括号 | 151 | `[qobuz] 标题`、`【Hi-Res】标题` |
| 碟号后缀 | 45 | `标题 CD1`、`标题 Disc 2` |
| 尾随年份 | 27 | `标题 2017` |
| 采样率/位深 | 20 | `标题 24-96`、`标题 FLAC1644` |
| 前置目录序号 | 3 | `01、标题` |

最终规则会：

- 删除锚定在名称边缘的日期、来源、格式、采样率、压制/发行版本和碟号噪声。
- 在已知歌手名被目录结构或稳定标签佐证时，删除专辑名前重复的歌手名。
- 统一全角字符、空白和繁体到简体，供展示与搜索使用。
- 只采用达到 **70%** 的非空标签多数；占位标签仍计入分母，占位值不能成为候选。
- 标签候选还必须能被原始文件夹名的规范化键包含，防止错误标签覆盖目录名。
- 保留 `1989`、`2001 A Space Odyssey`、`20 GREATEST HITS`、`No.1`、`24K Magic`、`J-GAME`、`AK-47`、`AC/DC`、`Blink-182`、`A-ha`、`家 III` 等有语义的数字、连字符和标点。
- 规则清空结果时回退到规范化原值；同一结果重复清洗必须保持不变。

## 最终覆盖率

生产实现对全部 4,867 张专辑进行了两种审计：一是构造强制封面状态，二是保留真实封面状态。名称规则现在与封面状态解耦，两条链路的结果应完全一致。

| 指标 | 结果 |
| --- | ---: |
| 原始专辑名经纯规则发生变化 | 4,613 |
| 全库展示链路专辑名发生变化 | 4,694 |
| 真实运行时展示专辑名发生变化 | 4,694 |
| 全库展示链路歌手名发生变化 | 249 |
| 稳定且被目录佐证的专辑标签匹配 | 3,876 / 3,876 |
| 稳定且被目录佐证的歌手标签匹配 | 4,348 / 4,348 |
| 空输出 | 0 |
| 幂等失败 | 0 |
| 清洗后仍被启发式标为结构噪声 | 10 / 4,458（0.2243%） |

残余 10 条全部属于“专辑名仍包含歌手键”的保守启发式命中，例如歌手同名专辑、`BEYOND IV`，以及歌手名本来就是完整标题的一部分。继续自动删除会误伤合法专辑名，因此将其保留；0.2243% 已低于本次设定的 2% 残余门槛。

真实运行时现在会让 4,584 张有封面专辑和 283 张缺封面专辑全部先经过确定性正则。缺封面状态及筛选结果保持不变；变化仅作用于展示名和搜索词。审计中的强制封面链路与真实封面链路均有 4,694 张专辑发生名称变化，证明封面状态已不再限制规则范围。

## 可复现审计

```shell
zsh Scripts/audit_album_name_cleaning.sh '<扫描快照.db>'
```

脚本直接编译并调用生产 `AlbumNameCleaning` 与 `AlbumDisplayNameCleaning`，而不是复制一套独立正则。输出包含快照规模、全部文件名结构、路径摘要、标签匹配、残余噪声、空输出和幂等性指标。
`qualityGate.passed` 只有在全部路径一致、标签匹配率至少 98%、残余噪声不高于 2%、空输出和幂等失败均为 0 时才会为 `true`。
