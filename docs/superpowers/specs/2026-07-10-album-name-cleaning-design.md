# 专辑展示名称清洗设计

## 目标

基于用户提供的完整扫描快照，完善专辑展示名和封面搜索词的确定性清洗规则。规则只生成派生的 `artistName` 与 `albumName`，不得修改扫描边界、真实路径、音频标签、`AlbumScanRecord` 原始字段或 SQLite/旧 JSON 快照中的原始数据。

## 数据证据

提供的 `.db` 实际为 schema v1 的旧 JSON 快照，兼容层会正常读取。全量遍历结果：

- 4,867 张专辑。
- 68,525 个音频文件；68,513 个含可读标签，12 个标签读取失败。
- 61,703 个文件名有明确曲目序号前缀，61,403 个文件名包含标签标题；曲目文件名适合作为核对证据，不适合被改写。
- 4,599 个专辑目录名命中至少一种可识别噪音；3,207 个有年份/日期前缀，3,983 个有括号组，3,049 个含格式词，1,736 个含版本词，528 个含码率，328 个含来源站点，328 个含碟号。
- 4,547 张专辑有 `metadata.album`；4,211 张存在占比至少 70% 的稳定多数标签。其中 4,052 张只有一个非空标签值，3,122 个单一标签名规范化后明确包含于目录名，可作为高置信辅助证据。
- 3,754 张专辑只有一个 `albumArtist`，4,256 张只有一个 `artist`；标签只在候选与目录歌手名相互印证时参与展示名选择。

## 方案

新增 Domain 纯规则 `AlbumNameCleaning`，把当前混用的一套规则拆成两个字段入口：

- `cleanArtistName`：繁转简、合并空白、删除已确认的目录容器/来源后缀。不得执行年份、碟号或专辑格式清洗。
- `cleanAlbumName`：按结构删除前置日期/列表编号/来源、明确重复的歌手前缀、尾部格式/码率/碟号/发行版本，并保留正式标题中的数字、横线、斜线和语义括号。

`AlbumDisplayNameCleaning` 对所有专辑统一执行，不以是否已有封面作为名称清洗门禁。对扫描原名进行清洗时，可读取音频标签形成辅助候选；仅当同一专辑内清洗后的标签候选占非空标签至少 70%、候选不是通用占位词、且候选规范键确实包含于目录原名时采用。Ollama 建议不被标签覆盖，只经过同一 Domain 纯规则后处理。

`AlbumNameSuggestionCleaner` 保留现有类型和调用方，但改为委托 Domain 规则，避免展示清洗和模型输出清洗继续漂移。

## 规则顺序

1. Unicode 宽度/空白归一化并繁体转简体。
2. 反复剥离开头的已知来源括号和独立日期括号。
3. 只在存在非空后缀时删除 `YYYY`、`YYYY-MM`、`YYYY-MM-DD`、`YYYYMMDD` 等前置日期；纯标题 `1989`、`2001` 保留。
4. 删除以 `、`、`．` 等明确列表符分隔的短编号壳。
5. 有歌手上下文时，只删除开头重复歌手及其明确分隔符；不得在正文中全局删除艺人名。
6. 单个书名号标题在其前后仅有歌手、唱片描述或技术噪音时提取正文；多个书名号表示组合发行时保留全部标题。
7. 从尾部向前反复删除已知噪音括号、发行日期、格式、码率、碟号和版本链。
8. 只清理边缘分隔符并合并空白；不得全局把 `-`、`/`、`.` 改为空格。
9. 清洗为空、只剩标点或没有可信结果时回退输入值。

被移出主标题的年份、版本、介质和碟号仍完整保留在扫描原名、详情路径和快照中；清洗结果不得用于合并同名专辑或重写身份。

## 防误伤契约

以下正式名称必须逐字保留核心标点和数字：

- `1989`
- `2001 A Space Odyssey`
- `20 GREATEST HITS`
- `No.1`
- `24K Magic`
- `J-GAME`
- `AK-47`
- `AC/DC`
- `2023VIP数字专辑(1)`
- `银色月光下(演唱会)`

原始 `relativePath`、标签和数据库内容必须保持不变。规则必须幂等，且不得返回空字符串。

## 覆盖率验收

覆盖率审计必须完整遍历 4,867 张专辑和 68,525 个音频文件，不以抽样代替：

- 数据遍历率：100%。
- 稳定多数标签且被目录名佐证的高置信样本中，清洗结果与标签核心候选的规范键一致率至少 98%。
- 命中明确结构噪音的目录名中，清洗后仍残留同类“前置日期/来源或尾部格式/码率/碟号”的比例不高于 2%。
- 空结果：0。
- 幂等失败：0。
- 表驱动正例和反例：100% 通过。

若残留率未达标，按残留模式频次补规则，同时为每条新规则增加一个反例；不得通过扩大任意位置的模糊删除来追求数字。

## 测试与验证

- `AlbumNameCleaningTests`：数据库提炼的纯规则正例、数字/标点/括号反例、幂等与空值回退。
- `AlbumDisplayNameCleaningTests`：有无封面统一清洗、标签仅辅助、冲突标签回退目录、歌手字段独立清洗。
- `AlbumNameEnhancementTests`：Ollama JSON 后处理复用同一规则。
- 完整数据库审计：本地只读运行，不提交 39 MB 快照或绝对路径。
- 最终运行 `git diff --check` 与 `zsh Scripts/verify.sh`。

## 自审

- 无待定项或占位符。
- 规则只影响展示和搜索，符合项目原始数据边界。
- 目录优先、标签辅助与覆盖率口径一致。
- 实现范围聚焦名称策略、对应测试和项目说明，不触碰当前有并行改动的 UI 文件。
