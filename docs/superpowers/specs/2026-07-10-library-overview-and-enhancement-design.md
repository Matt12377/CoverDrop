# 音乐库总览与智能解析范围设计

## 目标

减少封面墙顶部信息拥挤，令统计数量与当前筛选、搜索结果一致；智能解析改为由用户手动启动，并提供针对当前音乐库的停止控制。

## 顶部布局

- 左上区域依次显示音乐库名称、音乐库地址和当前动作内容。
- 删除“当前动作”标题，只保留动作内容，例如“正在智能解析 SACD”或“空闲”。
- 右上区域显示紧凑的智能解析进度，进度条固定为约 `240pt`，不再横跨页面。
- 进度条下方显示 `已完成数/总数`，例如 `93/350`，不重复显示动作内容。
- 没有可显示的智能解析进度时，不占用右上进度区域。

## 当前页面数量

- 顶部只显示一项数量，不再同时排列全部统计项。
- 数量标题跟随当前筛选标签：
  - “全部”显示“专辑数量”。
  - 其他标签使用标签名称，包括“缺封面”“未分轨”“标签异常”“track 音轨”“已有封面”“解析失败”“散落音频”。
- 数量值来自当前页面实际可见结果。
- 搜索关键词生效时，数量同步变为当前筛选页中匹配关键词的结果数。
- “散落音频”页统计匹配的散落音频路径数；其他页面统计匹配的专辑数。

## 智能解析边界与控制

- 音乐库级“智能解析”仅遍历传入 `libraryID` 对应扫描结果中的缺封面专辑。
- 已有封面的专辑不加入音乐库级手动解析队列，也不计入该次进度总数。
- 如果当前音乐库没有缺封面专辑，则结束请求并保持非运行状态，进度为 `0/0`。
- 专辑详情中的单张手动解析保持现有行为，仍允许用户显式解析已有封面的专辑。
- 初次扫描、重新扫描、加载扫描快照和实时局部刷新都不得自动启动音乐库级智能解析；扫描只更新扫描结果与快照。
- 用户只能通过音乐库操作菜单中的“智能解析”启动当前音乐库的批次。
- 在“智能解析”下方新增“停止解析”。它只在当前音乐库的智能解析批次正在运行时可点击，其他任何状态均置灰。
- 点击“停止解析”后，取消当前音乐库正在执行的请求并移除该音乐库剩余队列项；已成功生成的名称增强结果保留，当前被取消的专辑与未开始的专辑保持原始名称。
- 停止后的进度保留批次总数与已完成数量，但状态变为非运行；下次手动点击“智能解析”会重新建立缺封面专辑队列，已完成专辑可由既有队列去重/结果状态处理，不要求清除已完成结果。

## 数据流

1. `LibraryScanSummaryView` 使用当前筛选后的 `filteredAlbums` 或 `filteredLooseAudioPaths` 计算页面数量。
2. 顶部标题由当前 `AlbumScanResultFilter` 决定，不增加新的持久化状态。
3. `AppModel.scanLibrary(_:)` 与实时局部刷新仅保存/刷新扫描结果，不调用自动解析入口。
4. `AppModel.requestAlbumNameEnhancement(forLibraryID:)` 先筛选缺封面专辑，再建立队列和进度。
5. `AppModel.stopAlbumNameEnhancement(forLibraryID:)` 仅影响匹配 `libraryID` 的运行项和排队项，并把该库解析状态切换为非运行。
6. 进度 `totalAlbums` 使用实际成功加入队列的缺封面专辑数，避免 UI 与任务数量不一致。

## 错误与状态

- Ollama 未启用、扫描结果不存在等错误继续沿用现有中文提示。
- 解析失败仍记录到单张专辑状态，并通过“解析失败”筛选查看。
- 用户主动停止不是解析失败：不写入失败筛选，也不显示 Ollama 错误。
- 当前动作内容继续优先显示解析进度、实时刷新消息和失败摘要。
- 页面切换或搜索变化只影响显示数量，不改变扫描结果和解析队列。

## 验证

- 修改 AppModel 单元测试，证明扫描完成后不会自动启动解析，且音乐库级解析跳过已有封面专辑。
- 新增停止解析测试，证明停止会取消当前音乐库后续队列、保留已成功结果，并使状态回到非运行。
- 修改进度测试，证明进度总数只包含缺封面专辑。
- 保留单张手动解析已有封面专辑的回归测试。
- 运行应用，确认“停止解析”位于“智能解析”下方，并仅在运行中可点击。
- 运行 `git diff --check` 和 `zsh Scripts/verify.sh`。
- 运行应用并检查顶部布局，确认进度位于右上、宽度紧凑、下方显示 `xxx/xxx`，动作文字位于音乐库地址下方且没有“当前动作”标题。
